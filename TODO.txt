================================================================================
finna CLI — Implementation Plan
================================================================================

Goal: Go CLI `finna` for finna-app (FastAPI, deployed on GKE). Brew-installable.
Rich terminal output (tables, spinners, colors). TOML config. No full TUI.

Backend surface mapped from /Users/andrea/Projects/personal/finna-app/docs/openapi.yaml
Auth: JWT via /api/v1/auth/token (password), /api/v1/auth/login (CLI alias),
      GitHub OAuth, OIDC w/ PKCE, GCP/Azure service-account register.

PHASE 1 — Config + Context Management
================================================================================
Config file: $XDG_CONFIG_HOME/finna/config.toml  (default ~/.config/finna/config.toml)
Tokens: OS keyring (never on disk).

[ ] 1.1  TOML schema:
            current_context = "prod"
            [contexts.prod]
              server = "https://finna.<gke-domain>"
              insecure = false
              default_project = "..."
            [contexts.local]
              server = "http://localhost:8000"
            [ui]
              color = "auto"   # auto|always|never
              output = "table" # table|json|yaml|csv
[ ] 1.2  `finna context list|use|set|add|remove|current`  (kubectl-style)
[ ] 1.3  `finna config get|set|view|path`
[ ] 1.4  Global flags: --context, --server, --output (-o), --no-color, --debug
[ ] 1.5  Env overrides: FINNA_SERVER, FINNA_TOKEN, FINNA_CONTEXT, FINNA_OUTPUT
[ ] 1.6  First-run wizard: huh form → server URL → save context → suggest `finna login`

PHASE 3 — Auth
================================================================================
Token store key: "finna-cli:<context-name>" in OS keyring.

[ ] 3.1  `finna login`
            - default: POST /api/v1/auth/login (username/password via huh prompt)
            - --github : open browser → /api/v1/auth/github → poll callback
                         (or display URL + ask user to paste code)
            - --oidc <provider>: GET /api/v1/auth/oidc/providers → PKCE flow
                         → launch local loopback http server for callback
                         → POST /api/v1/auth/oidc/callback
            - --token <jwt>: store provided JWT directly
[ ] 3.2  `finna logout` (--all to wipe every context)
[ ] 3.3  `finna whoami` (decode JWT claims; show user, exp, provider)
[ ] 3.4  Auto-refresh: on 401, attempt re-login if refresh token present;
         else exit w/ "run `finna login`"
[ ] 3.5  `finna auth register gcp <file.json>`  → POST /api/v1/auth/gcp/register
[ ] 3.6  `finna auth register azure`            → POST /api/v1/auth/azure/service-account
[ ] 3.7  Admin OIDC provider mgmt:
            finna auth providers list|get|create|update|delete|test
